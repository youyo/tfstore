This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build / Lint / Test

```bash
go build ./...              # build all packages
go build -o tfstore .        # build the CLI binary
go vet ./...                 # static checks
go test ./...                 # run all tests
go test ./... -run TestName   # run a single test by name
go mod tidy                   # must produce no diff
```

Release builds go through GoReleaser (`.goreleaser.yml`), triggered by the
`goreleaser` GitHub Actions workflow (`.github/workflows/release.yaml`) on
tag creation. It cross-compiles for **linux/arm64 and darwin/arm64 only**
(no amd64) and publishes to GitHub Releases and a Homebrew formula in the
`youyo/homebrew-tap` tap (`brew install youyo/tap/tfstore`). The CLI version
is derived entirely from the git tag: GoReleaser injects it via `-X
main.version={{.Version}}`; do not hardcode a version constant elsewhere.

## Architecture

tfstore is a Go CLI that provisions a Terraform remote-state backend (an S3
bucket using Terraform's native S3 locking, `use_lockfile=true` — no
DynamoDB table) via a single CloudFormation stack.

- `main.go` — thin entrypoint; calls `cmd.Execute()` and exits 1 on error.
- `cmd/root.go` (package `cmd`) — the Cobra root command. Wires the
  `--stack-name/-n`, `--region/-r`, `--key/-k` flags and drives the
  New → Create → WaitForCreation → GetOutputs → BackendConfigExample flow
  via a package-level `newBackend` factory, which tests replace with a fake
  so the command can be exercised without AWS credentials or network access.
- `internal/backend/backend.go` (package `backend`) — the `Backend` type
  wraps a narrow CloudFormation client interface (`CFNAPI`: `CreateStack` +
  `DescribeStacks`, satisfying `cloudformation.DescribeStacksAPIClient` so it
  plugs straight into SDK waiters). Exposes `New`, `Create`,
  `WaitForCreation`, `GetOutputs`, and `BackendConfigExample`. `Create`
  always passes `internal/cfn.Template` as `TemplateBody` — this is a fixed
  contract, not an implementation detail.
- `internal/cfn/template.go` + `internal/cfn/cloudformation.yaml`
  (package `cfn`) — the CloudFormation template lives in its own YAML file
  and is embedded via `//go:embed` into the exported `Template string`. Do
  not rename or retype this symbol; `internal/backend` depends on it
  directly.
- `completions/_tfstore` — zsh completion script (generated by
  `go run . completion zsh`), bundled into release archives.

AWS credentials/region come from the standard SDK v2 default config chain
(`config.LoadDefaultConfig`), so no explicit config file is needed by the
tool itself.

## Testing policy (TDD)

Every behavior-bearing change follows Red → Green → Refactor. Tests use the
standard library only — no mocking framework, no `httptest`-based AWS
fakes:

- AWS calls are isolated behind narrow interfaces (`backend.CFNAPI`,
  `cmd.backendAPI`) so tests inject function-field fakes directly; nothing
  touches real AWS credentials or the network.
- `backend.BackendConfigExample` has a golden-string test — its output must
  stay byte-for-byte in sync with the example in README.md.
- `internal/cfn`'s test asserts on the embedded template's textual content
  (must contain S3/PublicAccessBlock/DeletionPolicy, must not contain
  DynamoDB or `Transform`) rather than parsing CloudFormation.

## Dependency policy

Keep the dependency set intentionally small and pinned in `go.mod`/`go.sum`:

- `github.com/aws/aws-sdk-go-v2` + `.../config` + `.../service/cloudformation`
  — AWS access.
- `github.com/spf13/cobra` — CLI framework. **No viper**: a single
  create-only command needs flags plus the AWS credential chain, not a
  layered config system.

Do not add a new direct dependency without a concrete need; prefer stdlib.
`go mod tidy` must produce no diff.
